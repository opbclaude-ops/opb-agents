Тонкая обёртка над навыком `spec-collect` v1.0. Источник истины — `SKILL.md` навыка. Здесь — только транспорт.

## Шаги

1. **Понять вход.** Что прислал пользователь:
   - ссылка/id папки Drive,
   - локальный путь к PDF или папке проекта (`/mnt/user-data/uploads/...`),
   - приложенный PDF.

2. **Определить режим.**
   - «Одна система» — один PDF + название системы, либо явное «собери {название}».
   - «Все системы» — папка проекта с подкаталогами/файлами по системам.
   - Если по сообщению неясно — один уточняющий вопрос.

3. **Скачать материалы в `/home/claude/<имя_проекта>/`.**
   - Для Drive: `search_files` по родителю → `download_file_content` нужных PDF.
   - Проверить, что в чате есть xlsx-выгрузка `База_3_ОПБ` (id `1WisK24J9LHPTAVumkNsL2M-S6O38EwJ1c0P8Em45QMQ`). Без неё матчинг номенклатуры невозможен — попросить пользователя выгрузить (Файл → Скачать → Microsoft Excel).

4. **Убедиться, что навык `spec-collect` доступен.** Порядок поиска:
   1. Локально: `~/.claude/skills/spec-collect/` или `/mnt/skills/user/spec-collect/`.
   2. Fallback с Drive: папка `Skills/spec-collect/v1.0/` (родитель `Skills` — id `1Qh8-R0okEZTPF_zHVRUPGCRhwSh7jqQu`). Выгрузить через `download_file_content`, восстановить структуру в `/home/claude/.skills/spec-collect/` (SKILL.md, references/, scripts/, assets/).

5. **Прочитать `SKILL.md` навыка целиком.** Дальше — строго по нему: 9 этапов с воротами (режим → идентификация системы → парсинг PDF → сегментация → классификация Тип → матчинг номенклатуры → заполнение 11 колонок → самопроверка → запись листа → финальный лист «Все системы»).

6. **Запустить навык.** Прогнать скрипты `parse_pdf_spec.py` / `parse_xlsx_spec.py` / `match_nomenclature.py` / `build_output.py` ровно как описано. Не вмешиваться, не сокращать самопроверки, не догадываться о номенклатурной группе при низкой уверенности — флаг «требует ручной проверки», как требует навык.

7. **Отдать через `present_files`.** Краткая сводка: какие системы вошли, число строк в каждом листе, флаги «требует ручной проверки», какие позиции не прошли самопроверки и почему.

## Принципы

- Навык — источник истины, агент — транспорт. Хочется поправить логику — правится `spec-collect`, не этот файл.
- Триггеры в `description` агента = триггеры из `description` навыка.
- Самопроверки навыка (ворота на каждом этапе) не дублировать и не обходить.
- Колонки «Индекс наряда / Наряд / СМР ОПБ» в выходе — пустые. Это работа других агентов.
- Версия привязана: `agent-spec v1.0` ↔ `spec-collect v1.0`. Появится `v1.1` навыка — обновим обёртку.
