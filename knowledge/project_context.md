# Контекст проекта `workproduction`

> Базовые правила, термины, стоп-зоны. Читают **все агенты** проекта при
> старте. Не дублировать в теле каждого агента — ссылаться сюда.

## Кто пользователь

- **Сметчик ОПБ** (отдел подготовки бюджетов) компании **Авентин**.
- Считает наряды на монтаж слаботочки **после выигрыша тендера** —
  производственный расчёт для оплаты бригадам.
- Аккаунт Drive: `opbclaude@gmail.com` (отдельный, под работу с Claude).
- С телефона Samsung S20 FE — короткие сообщения, иногда с опечатками.

## Системы в скоупе

АППЗ, СВН, ТСО, СКУД, СОУЭ, ДП, ЭОМ. Объекты — ЖК и паркинги.

## Терминология

- **Единая расценка** — одна цена работы, общая для ОПБ и КД. Лист
  «Справочник» в файле «Единые расценки работ Авентин».
- **ВСП** — вспомогательные работы (штроба, отверстия, узлы, ПНР). Не
  входят в наряды напрямую, считаются по методике.
- **ПНР** — пусконаладочные. Часть ВСП-семейства, для АППЗ — 22 строки
  методики из листа «Методика» в `База_3_ОПБ`, секция «5. ПНР АППЗ (РУБЕЖ)».
- **Тождество корректности**: единая расценка по спеке + ВСП по методике
  = сумма фактических нарядов из **Синтеки**. Для АППЗ точность 1,02–1,05.
- **Синтека** — учётная ERP Авентина, источник истины, **есть API**.
- **Пакет** — наряд с сопутствующими работами в одной строке (штроба +
  гофра + кабель + монтаж).
- **Классификатор оборудования** — компактный реестр «марка → группа →
  наряд». Растёт из практики.
- **BTH** — адресный извещатель (ИП 212-64-R3), обычно в прихожей.
- **BTHA** — автономный извещатель (ИП 212-142), во всех жилых
  помещениях + прихожая. Количество = жилые помещения + 1.

## Грабли (накоплено за несколько чатов)

1. **Не лить «маркер с метаданными» вместо полного дампа.** Если дамп
   превысил безопасный потолок (80 KB на один `create_file` вызов) —
   сказать пользователю и получить согласие, не подменять молча.
2. **5 BTHA — двойственно.** Может быть 2-комн евро (50–65 м²) ИЛИ
   3-комн классика (75+ м²). Без плана/площади BTHA-метод ошибается.
3. **Опорная квартира при квартирографии — однушка или двушка,
   НЕ студия.** На студии 2 BTHA двойственно.
4. **В таблице квартирографии евро- и классические форматы не
   разделены.** Двушка-евро и двушка-классика обе идут в столбец «2к».
5. **Подписи типов в квартирах на плане (М3.1) — приоритет №1** для
   квартирографии. Если есть — это финал. BTHA-счёт (М2.1) — резерв.
6. **Бэкап навыков на Drive — только текстовый `.md`,** не бинарь (через
   base64 ловит классификатор безопасности).
